# UniGest Demo Release

This document defines the demo build we can show safely before production.

## Demo Goal

Show the complete ESSE3/Alice-like student lifecycle without requiring real university data:

- Student dashboard, career/libretto, exams, grades, fees, certificates, thesis/graduation.
- LMS course player with progress, announcements, assignments, quizzes, and forum.
- Internships and alumni/placement flows.
- Admin reporting, audit, internships, alumni, and back-office modules.
- Teacher course, exams, verbale, and LMS authoring flows.

## Demo Mode

The web app already supports `NEXT_PUBLIC_DEMO_MODE=true`.

When enabled:

- auth middleware is bypassed;
- a floating role switcher appears;
- roles available: student, teacher, admin;
- demo users are generated by `packages/web/lib/supabase/server.ts`;
- routes remain close to production navigation while avoiding real credentials.

Use `.env.demo.example` as the demo environment reference.

## Local Demo Run

Recommended local sequence:

```bash
npm install
npm run verify:demo
npm run dev:web
```

Then open:

- Web: `http://localhost:3000`
- Student: `http://localhost:3000/student/dashboard`
- Teacher: `http://localhost:3000/teacher/dashboard`
- Admin: `http://localhost:3000/admin/dashboard`

For API-backed demo data, run:

```bash
npm run dev:api
```

## Demo Script

1. Start on student dashboard.
2. Show quick access: libretto, exams, fees, certificates, thesis, internships, alumni.
3. Open LMS courses:
   - course progress;
   - content completion;
   - announcements;
   - assignment submission;
   - quiz attempt;
   - forum reply.
4. Switch to teacher:
   - course/LMS editor;
   - assignments/submissions;
   - quiz management;
   - exam/verbale flow.
5. Switch to admin:
   - reports;
   - audit logs;
   - internships;
   - alumni placement;
   - fees/certificates.

## Demo Safety

Do not use real students, real grades, real payment refs, or production Supabase keys in demo mode.

Required demo flags:

- `NEXT_PUBLIC_DEMO_MODE=true`
- `REQUIRE_MFA=false`
- no production `SUPABASE_SERVICE_ROLE_KEY`
- no production email sender unless explicitly testing email

## Demo Acceptance Checklist

- `npm run verify:demo` passes.
- Web app starts with role switcher visible.
- Student, teacher, and admin dashboards render.
- LMS mobile/web flows have been type-checked.
- API unit tests pass.
- No `.env.local` or real secrets are committed.
- Demo URLs and credentials are documented for presenters.

## Known Demo Limits

- Mobile demo still requires a real Expo/Supabase configuration for authenticated device testing.
- Push delivery requires deployed Supabase Edge Function and Expo push credentials.
- Some demo screens use static/mock data when `NEXT_PUBLIC_DEMO_MODE=true`.
